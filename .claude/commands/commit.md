Analyze the staged changes and create a git commit following Conventional Commits:

- Format: `type(scope): description`
- Types: feat | fix | refactor | chore | docs | test | perf | ci
- Subject line max 72 characters, imperative mood
- Add a body if the change needs context
- Do NOT include any co-authored-by or attribution lines
